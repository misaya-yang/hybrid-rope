# 统一实验协议（冻结版）

更新时间：2026-02-23  
作用：把“可比性”写死，避免再次出现“算力花了但结果不可用”。

## 1. 总原则（必须同时满足）

1. 同一结论内所有方法必须共用同一训练和评测协议。  
2. RoPE 注入只允许一种机制：`inv_freq` 原地覆写。  
3. 禁止在同一对比组中混用 `rope_scaling` API 与 monkey patch。  
4. 每次实验必须记录元信息：tokenizer、dataset、seed、脚本版本、时间戳。  
5. 任何 `PPL` 异常到不符合模型规模常识时，先判为“协议异常”，不做结论。

## 2. RoPE 注入红线

适用脚本：`2026-02-22/scripts/run_llama8b_fair_suite.py`

必须：
1. `inv_freq.copy_(custom_inv_freq)` 覆写。  
2. 覆写后清空 rotary cos/sin cache。  
3. 保持一致 attention 实现（例如统一 `sdpa`）。  

禁止：
1. 修改 `forward`（monkey patch attention forward）。  
2. 一组对比里有人用 `config.rope_scaling`，有人用自定义注入。

## 3. 数据与 tokenizer 冻结

## 3.1 允许的数据模式

1. 论文主线训练：真实语料（TinyStories / LongBench 本地镜像等）。  
2. `Synthetic-Passkey` 仅允许用于 sanity，不可用于主结果。  

## 3.2 tokenizer 规则

1. 论文主线禁止 silent fallback 到 byte tokenizer。  
2. 若出现 byte fallback，实验立即失败并记录。  
3. 元信息必须输出：`tokenizer name + vocab_size + token_id_range`。

## 3.3 针对本仓库已加的防呆

脚本：
1. `sigmoid_rope_experiments/run_phase4.py`  
2. `sigmoid_rope_experiments/run_phase4_corrected.py`

已加保护：
1. 非 byte 模式若落到 byte tokenizer，直接报错。  
2. 非 synthetic 模式若落到 synthetic dataset，直接报错。

## 4. 评测协议冻结

## 4.1 PPL

1. 明确记录序列长度与样本数。  
2. loss 必须按 token 归一化，不能按 batch 简化。  
3. 对外推长度（16K/32K）要与历史口径一致。

## 4.2 Passkey

1. 对 base 模型优先 teacher-forcing 对比（true suffix loss vs false suffix loss）。  
2. generation-only 结果不作为唯一判断。  
3. 输出命中率与样例 loss 对照，防止“全零/全一假象”。

## 4.3 NIAH / LongBench

1. 评测脚本固定版本、固定 prompt 模板。  
2. 若中途切换模板，结果自动降级为不可比。  
3. 每个结果都要对应原始 JSON/CSV。

## 6. 结果入库门禁（必须全绿）

每次新结果写入论文前，必须满足：
1. 元信息完整：`summary.json`/`run_meta.json` 存在。  
2. 协议一致：tokenizer/dataset/注入机制符合本文件。  
3. 数值合理：不出现明显尺度异常（如 124M 出现 `PPL≈1` 且协议不一致）。  
4. 有可追溯路径：结果文件可从仓库相对路径定位。
