## 4. 协议防错（必须执行）

## 4.1 结果门禁

任何结果进入论文前，必须检查：
1. `summary.json` 或 `run_meta.json` 是否存在。  
2. tokenizer 不是 byte fallback（除非实验就是 byte 对照）。  
3. dataset 不是 synthetic fallback（除非实验就是 synthetic 对照）。  
4. 数值尺度是否符合模型规模常识。

## 4.2 停机止损条件

出现以下任一情况立刻停：
1. 日志持续 NaN。  
2. 明显协议漂移（例如主线 run 使用了 synthetic+byte）。  
3. 对比组注入机制不一致（公平性破坏）。

## 5. 产出归档规范

每次完成后执行：
1. 把 `results/overnight_8h/` 下 `summary.json`、`logs`、`artifacts` 拉回本地归档。  
2. 更新 `handoff_2026-02-23/01_结果真值表_VALID_INVALID.md` 的状态位。  
3. 禁止直接覆盖旧结果目录，必须新建带时间戳目录。

## 6. 一条可执行的最小流程（推荐）

1. 跑理论三图（30~90 分钟）。  
2. 启动 overnight 8B 公平套件。  
3. 第二天按真值表判定 `VALID/PENDING/INVALID`。  
4. 只有 `VALID` 结果进入论文主表。
