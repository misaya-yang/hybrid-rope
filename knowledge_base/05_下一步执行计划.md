# 下一步执行计划（论文冲刺版）

## 1. 总目标

把当前“多条证据并行但口径分散”的状态，收敛成一套可投稿的统一证据链：

1. 方法有效性：长程指标稳定改进。  
2. 任务有效性：检索与问答不崩。  
3. 机制可解释：能解释成功与失败。  
4. 工程可复现：脚本、参数、结果一一对应。

---

## 2. 48 小时计划（必须完成）

## 2.1 任务 A：完成训练期验证闭环

目标：完成标准频率 vs Sigmoid/Hybrid 的同条件训练比较。

交付：

1. 训练曲线（train/val loss）。  
2. 训练结束指标表（final + best checkpoint）。  
3. 基础结论：谁在验证集更优，差距多大。

判据：

- 两条线都完整跑完。  
- 日志无中断。  
- 指标口径一致。

## 2.2 任务 B：统一后评测协议

目标：同一模型组上统一执行 PPL + NIAH + LongBench。

交付：

1. 三类指标汇总表。  
2. 指标冲突分析（如果存在）。  
3. 任务失败样例摘要（用于论文误差分析）。

判据：

- 不允许只报某一类指标。  
- 不允许混用不同提示词协议。  
- 不允许不同长度使用不同解码参数。

## 2.3 任务 C：修复验证（高频保护 + 模板一致）

目标：验证“高频保护”和“指令格式一致”是否能修复 8B 冲突。

交付：

1. 修复前后对照表。  
2. 至少一个下游任务明显回升。  
3. 失败配置保留并解释。

判据：

- 修复策略可重复。  
- 改善方向一致。  
- 不是只在单长度/单任务生效。

## 6. 验收标准（Go / No-Go）

达到以下条件才进入“可投稿”状态：

1. 核心结论至少两次复现。  
2. PPL 和下游任务指标方向一致或可解释。  
3. 负结果与修复路径完整呈现。  
4. 图表和数字在文档中可闭环追溯。

如果任一条件不满足，继续迭代，不进入最终投稿稿件。

---

## 7. 给后续 AI 的执行准则

1. 先补闭环，不要先拓展。  
2. 先修协议，不要先调参数。  
3. 先保留失败，不要只留成功。  
4. 先做可复现，不要追单次最优。

按这四条执行，能最大化算力投入产出比。
