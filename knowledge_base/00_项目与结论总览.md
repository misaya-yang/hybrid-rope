# 项目与结论总览

> 最后更新：2026-02-22（新增公平 8B 实验设计与 overnight 流水线）

## 1. 项目定位

围绕长上下文位置编码的系统性研究，建立三层闭环：

1. **现象层**：频谱形状改造在长长度下确实比标准方法更稳。
2. **机理层**：Phase Collision 指标量化"为什么更稳"。
3. **工程层**：公平训练协议 + 多指标评测 → 结论可复现。

## 2. 核心研究问题

1. 通过重排 RoPE 频率分布，能否系统性延缓长程崩溃？
2. 这种改动在指令模型中是否会牺牲任务能力，若会，如何通过高频保护修复？

## 3. 证据体系（按强度排序）

### Tier 1：强稳定证据（从零训练）

| 规模 | Geo 500k PPL@16K | Hybrid PPL@16K | 改善 |
|------|----------------:|---------------:|-----:|
| 50M (3cfg×3seed) | 18.207±0.768 | **17.324±0.360** | ~5% |
| 100M | 10.888 | **9.417** | ~13.5% |
| 350M | 14.653±3.851 | **12.646±3.093** | ~13.7% |

跨规模一致正向改善，是论文最可信主线。

### Tier 2：机理实验（推理侧）

- Geo(10k) 崩溃比 22x → Sigmoid(100k) 崩溃比 1.08x
- Phase Collision 平均改善 ~55.67%（Sigmoid 公式拟合 R²>0.99）

### Tier 3：8B LoRA（进行中 🔄）

**历史问题**（已识别并修复）：
旧实验 YaRN/PI 用 HF `rope_scaling`，Hybrid 用自定义 monkey patch → 不公平。

**新设计**（今晚 overnight 实验）：
- 统一使用 `inv_freq.copy_()` buffer 覆写，禁止 `rope_scaling` 和 `forward` patch
- 4 方法（baseline / PI / YaRN / anchored_hybrid）完全同条件
- 训练 + NIAH 评测一体化流水线
- 脚本：`2026-02-22/scripts/run_overnight_8h.py`

### Tier 4：负结果（工程价值）

- Zero-shot 直接替换频率 → 大概率崩溃
- PPL 改善 ≠ 下游任务改善（8B 旧实验已验证）
- 频谱形状 vs base 选择：V2 实验显示统一 base 后 geo 在 phase collision 上最优 → 需要在训练实验中验证 shape 是否仍有优势

## 4. 当前最可信结论

1. ✅ 频谱形状设计在长程外推上是有效方向（从零训练已证）
2. ✅ 频谱形状是独立于 theta 放大的关键变量
3. ✅ 高频保护是维持指令能力的必要条件
4. ⏳ 公平 8B 协议下哪种频谱方案最优（tonight 实验将回答）

## 5. 暂不宜直接宣传

1. "所有协议下都优于 YaRN/PI"
2. "无需训练可直接替换频率"
3. "单个 PPL 指标改善即代表方法有效"

## 6. 论文结构建议

1. **问题**：标准频谱在长程发生相位冲突
2. **方法**：重构频谱形状 + 高频锚定保护
3. **证据**：从零训练多规模 + 公平 8B LoRA + 机理指标
4. **结论**：给出"何时有效、何时失效、如何修复"的边界条件
