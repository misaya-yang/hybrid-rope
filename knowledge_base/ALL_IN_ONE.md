## 1. 研究背景与核心问题

长上下文语言建模有两个常见症状：

1. 长度超出训练区间后，困惑度（PPL）显著恶化，通常呈指数式上升。  
2. 即使 PPL 看起来可控，下游任务（检索、问答、摘要）也可能突然退化。  

本项目的核心目标是：

1. 用频率分配设计（而非改模型主体）提升长度外推稳定性。  
2. 在长程改进的同时尽量保持短程能力与指令能力。  
3. 给出可解释、可复现、可迁移的参数规律（尤其是 Sigmoid-RoPE 公式化）。

## 2. 方法直觉与理论框架

### 2.1 标准 RoPE 的限制

标准几何频率在训练长度附近通常工作良好，但在更长距离会出现相位密集重叠，导致远距离区分度下降。  
直觉上，模型会越来越难分辨“远处 token 的相对位置差异”。

### 2.2 本项目的方法方向

本项目探索了三类“频谱重分配”思路：

1. Hybrid：几何频率与多项式/非线性频率混合。  
2. Anchored/分段锚定：锁住高频通道，只重塑低频通道。  
3. Sigmoid：把离散几何比率改为连续可控曲线，使频谱在高频端与低频端更平衡。

### 2.3 关键理论指标

除了 PPL，本项目强调一个机理代理指标：Phase Collision。  
直观含义是：不同相对距离的相位表达是否互相“打架”。  
指标下降意味着长程位置区分度更好。

### 2.4 重要方法学结论

1. 仅增大 theta 不是充分条件。  
2. 高频通道是短程语法与指令格式的敏感区，随意改动高频往往导致灾难性副作用。  
3. 频谱方法在“推理时即插即用”与“训练时协同适配”之间有本质差异。

## 4. 核心实验结果（详细版）

## 4.1 从零训练主线（最稳定、最可复现）

### 4.1.1 50M（三配置三种子）

| 配置 | PPL@2048 (mean±std) | PPL@16384 (mean±std) |
|---|---:|---:|
| Geo 500k | 6.826 ± 0.048 | 18.207 ± 0.768 |
| Hybrid a0.2 t100k | 6.699 ± 0.168 | **17.324 ± 0.360** |
| AnchPoly p3.9 | 6.634 ± 0.192 | 19.133 ± 1.135 |

解读：

1. 50M 的 16K 指标上，Hybrid 相比 Geo 有稳定优势。  
2. AnchPoly 在这组设置中不如 Hybrid。  
3. Hybrid 的方差可控，没有靠偶然 seed 取胜。

### 4.1.2 50M（二配置十种子）

- Geo 500k：PPL@16384 = 18.798 ± 1.643  
- Hybrid a0.2 t100k：PPL@16384 = 17.777 ± 2.824

解读：

1. 均值仍然支持 Hybrid。  
2. 但 Hybrid 方差更大，提示参数和训练噪声敏感性更高。

### 4.1.3 50M 频谱公平因子实验（6 配置）

PPL@16384（mean±std）：

- geo_100k: 11.904 ± 0.473  
- geo_200k: 18.524 ± 3.298  
- geo_300k: 13.797 ± 1.127  
- geo_500k: 13.757 ± 0.348  
- hybrid_a0.2_t100k: 13.508 ± 1.100  
- hybrid_a0.2_t500k: 13.495 ± 1.005

解读：

1. 单纯放大 theta 并不总是单调更好。  
2. 频谱形状和 theta 存在交互项。  
3. Hybrid 在公平设置下可达到最优或近最优区间。

### 4.1.4 50M YaRN 对照（修正版评测）

| Length | Hybrid(native) | Geo(native) | Geo+YaRN(progressive) |
|---|---:|---:|---:|
| 2048 | 6.672 | 6.839 | 6.839 |
| 4096 | 6.748 | 7.045 | 8.640 |
| 8192 | 8.688 | 8.833 | 16.899 |
| 12288 | 13.333 | 13.588 | 29.352 |
| 16384 | **16.861** | 17.966 | 39.479 |

解读：

在这组 50M 协议中，Hybrid > Geo > Geo+YaRN(progressive)。

### 4.1.5 100M 与 350M 的跨规模验证

100M：

- Geo 500k：10.888  
- Hybrid a0.2 t100k：9.417  
- 相对改善：约 13.5%

350M：

- Geo 500k：14.653（std 3.851, n=10）  
- Hybrid a0.2 t100k：12.646（std 3.093, n=10）  
- 相对改善：约 13.7%

总解读：

从 50M 到 350M，Hybrid 的 16K 优势保持一致，这是当前最可靠的证据链之一。

## 4.3 8B LoRA 主线（最有争议也最有价值）

### 4.3.1 强阳性结果

同一组实验里：

- 2K：15.889 vs 15.352  
- 8K：13.633 vs 13.506  
- 16K：190.566 vs **15.400**

16K 降幅约 91.9%，是本项目最“惊艳”的单点结果。

### 4.3.2 机制 2x2 因子分解（16K）

- base + 原频率：190.566  
- base + hybrid频率：7612.143  
- LoRA + 原频率：231.308  
- LoRA + hybrid频率：15.400

关键解释：

1. 频率和 LoRA 的组合效应极强。  
2. 只改频率或只做 LoRA 都可能失败。  
3. 成功不是“任意频谱替换”带来的，而是耦合条件下出现。

### 4.3.3 公平 8B 协议（600 step）

| 变体 | PPL@16K | PPL@32K |
|---|---:|---:|
| YaRN | **6.057** | **6.270** |
| PI | 6.137 | 6.310 |
| Hybrid | 11.875 | 77.138 |
| PI-soft | 218.624 | 2958.352 |

解读：

这组结果和强阳性样例冲突，说明当前 8B 线并未形成“稳定闭环”。

### 4.3.4 下游评测冲突（NIAH + LongBench）

NIAH 单针：

- Base：4K/8K 基本 1.0，16K/32K 近 0。  
- Hybrid-LoRA：大多数格点为 0。

LongBench（3 任务）：

| 任务 | Base | Hybrid-LoRA | 变化 |
|---|---:|---:|---:|
| qasper(F1) | 0.0699 | 0.0083 | -0.0616 |
| hotpotqa(F1) | 0.0114 | 0.0041 | -0.0073 |
| gov_report(R-L F1) | 0.1364 | 0.0369 | -0.0994 |

解读：

“PPL 改善”和“任务能力改善”并不自动等价。当前必须把两者统一优化。

## 4.5 Sigmoid 机理主线（Phase2/Phase3）

### 4.5.1 拟合结果

Phase2 拟合得到过：

- k 近似 `10 * L^0.047 / d`  
- x0 近似 `0.475 * d`

Phase3 模型选择进一步得到：

- k 近似 `16.223 / d`（R² 约 0.992）  
- x0 近似 `0.4658*d + 0.1713*ln(L)`（R² 约 0.999）

### 4.5.2 指标收益

- 公式参数下，phase collision 平均改善约 55.7%。  
- 最好改善约 66.5%，最小改善约 45.1%。  
- 公式参数与最优网格参数平均差距约 1.9 个百分点。

### 4.5.3 鲁棒性

在 d=128、L=131072 的敏感性分析中：

- k 的有效区间约 0.0655 到 0.25。  
- x0 的有效区间约 37.29 到 60.8。  

解读：

参数并非“针尖调参”，存在可操作鲁棒区间，这对工程落地很重要。

## 6. 目前最可信主张（可对外）

1. 在从零训练路线上，频谱设计可稳定改善长程外推。  
2. 频谱形状是独立有效因素，不是简单增大 theta。  
3. 8B 指令模型下，频率方法对训练协议和数据格式高度敏感。  
4. Sigmoid 在机理指标上成立，但训练期与下游闭环仍在完成中。

## 8. 冲突结果的统一解释框架

当前冲突（例如 16K PPL 强阳性 vs NIAH/LongBench 退化）可统一解释为三类问题：

1. 高频通道污染：局部句法与模板能力被破坏。  
2. 指令遗忘：训练数据形式和任务形式不一致。  
3. 注入实现偏差：缓存、方向、dtype、边界长度处理不一致。

这三类问题都属于“可修复工程问题”，而不是“频谱思想无效”。
