# NeurIPS Reviewer-Oriented Audit: Joint Attention Optimization LoRA (2026-02-26)

## 1) Reviewer-Critical Risks (and what was fixed)

1. `Data-source fragility`  
- Risk: long-task corpus missing causes run failure or hidden fallback to weak data.  
- Fix: training script now supports:
  - local `LongAlpaca/LongQA` path,
  - optional mirror URL fetch,
  - optional DashScope bootstrap,
  - explicit WikiText-long fallback (toggleable), all recorded in `run_config.json`.

2. `Attention regularizer not actually active`  
- Risk: `sdpa` often does not return attentions for custom penalties; method degrades to CE-only silently.
- Fix: when attention regularization is enabled, script auto-switches to `eager` if `attn_implementation=auto`.

3. `Transformers version drift`  
- Risk: `TrainingArguments` / `Trainer` API mismatch causes runtime crash (`evaluation_strategy` vs `eval_strategy`, `tokenizer` vs `processing_class`).
- Fix: added runtime signature compatibility logic.

4. `Unstable joint loss at early steps`  
- Risk: macro/micro regularizers dominate too early, harming instruction quality.
- Fix: added regularizer warmup (`regularizer_warmup_steps`) and logging fields:
  - `loss_micro`, `loss_macro`, `regularizer_scale`.

5. `Insufficient adapter capacity for joint objective`  
- Risk: rank too low under joint optimization underfits schedule + regularizer constraints.
- Fix: default rank for joint pipeline uses `r=32` and broader target modules
  (`q,k,v,o,gate,up,down`).

6. `Attention patch runtime shape bug`  
- Risk: patched attention occasionally inferred `num_heads=1`, causing reshape crash.
- Fix: robust head-dimension inference from projection shapes; added closure capture fix to avoid late-binding bugs.

7. `Reference prior accumulation bug`  
- Risk: histogram accumulation failed when warmup batches had different sequence lengths.
- Fix: histogram tensors now auto-pad to max length before accumulation.

8. `Collator incompatibility on labels`  
- Risk: pre-written variable-length `labels` field caused DataCollator tensorization failure.
- Fix: tokenized dataset now only stores `input_ids/attention_mask`; labels are generated by `DataCollatorForLanguageModeling`.

## 2) Current Recommended Training Standard

- Base model lock: `Meta-Llama-3-8B-Instruct` (8K-native only)  
- Steps:
  - smoke gate: 40 steps
  - full run: 800 steps
- LoRA:
  - `rank=32`, `alpha=64`
  - target modules (current strict controlled run): `q_proj,k_proj,v_proj,o_proj`
- Joint optimization (production-safe default):
  - attention bias: `bias`
  - `gamma=3e-4`
  - `max_seq_len=8192`, `batch_size=2`, `grad_accum=1`, `attn_impl=sdpa`
  - this is the current 96GB single-GPU steady point
- Joint optimization (advanced optional ablation):
  - attention bias: `bias+gate`
  - optional macro/micro KL
  - `lambda_micro=0.005`, `lambda_macro=0.003`
  - warmup: `regularizer_warmup_steps=80`
  - run only after base pipeline is stable

## 4) Why these settings are defensible (paper-facing)

- `Rank 32` is a compute-efficient middle point:
  - stronger than minimal-rank adaptation,
  - still affordable on single 96GB GPU under 8K–16K sequence settings.
- `800 steps` for this pipeline balances:
  - enough signal for joint objective to take effect,
  - avoids heavy overfitting risk and compute waste from very long low-quality runs.
- `Warmup regularizer` addresses the common reviewer concern:
  - “is gain from stable optimization or from accidental regularization dominance?”
  - warmup makes optimization behavior auditable and reproducible.
