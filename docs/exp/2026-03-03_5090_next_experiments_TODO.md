## 背景问题

### 问题 1：Passkey 对小模型是垃圾指标
- 350M 多 seed 数据显示 retrieval ~55%（Geo 和 EVQ 都是），纯噪声
- FineWeb-Edu 里没有任何 passkey 模式，小模型做 passkey 是 zero-shot
- 100 个 seed 下都是噪声，这个指标对 from-scratch 小模型完全无效
- **结论**：当前实验框架下 passkey 不可用，需要改变

### 问题 2：Geo 长程退化是正常现象
- R6000 750M Geo 50%→75%：PPL@8K +9.4%, PPL@16K +10.7%
- 训练在 L=2048 上，模型自然会越来越拟合 2K，长程外推退化是预期行为
- EVQ 会不会也退化？等 R6000 的 Hybrid checkpoint 结果（已在跑，不需要 5090 重复验证）

### 核心思路
5090 应该做 R6000 做不了的事，而不是重复 R6000 的 checkpoint trajectory。

## 方案 A：混合人工 Passkey 数据训练（⭐ 优先考虑）

**想法**：在 FineWeb-Edu 训练数据中混入少量人工构造的 passkey/retrieval 样本，让模型学会检索，然后对比 Geo vs EVQ 在"有检索能力"的情况下谁更强。

**动机**：
- 当前 passkey 无效是因为模型根本没学过检索
- 如果给模型检索能力后，EVQ 的频率重分配是否能让检索更准？
- 这分离了两个变量：①模型是否有检索能力（训练数据决定） ②频率分配是否帮助检索（EVQ vs Geo）

**实验设计**：
- [ ] 构造人工 passkey 训练样本（格式：长文本 + 隐藏 key + 问答）
- [ ] 确定混合比例（1%? 5%? 10%?）— 太多会污染 PPL 评估
- [ ] 350M, L=2048, base=500K, seed=42
- [ ] 对比组：Geo vs EVQ(τ=1.5)，同样的混合数据
- [ ] 评估：PPL@多长度 + passkey retrieval

**待回答的问题**：
- [ ] 混合比例多少合适？太多影响 PPL 的可比性
- [ ] 人工样本怎么构造？简单的 "The passkey is XXXXX" 还是更自然的？
- [ ] 会不会引入 confound：模型学到了检索模式，但这跟 PE 无关？
- [ ] 如果 Geo+混合数据也能 100% passkey，那说明 passkey 不区分 PE 方法
- [ ] 训练 tokens 多少？50M 够不够？

**风险**：
- 混合数据可能让两边都学会 passkey → 无法区分 → 白做
- 但如果 EVQ 在更长的 context（如 8K, 16K）上 passkey 更好，那就是强信号
- 这个实验的价值取决于"检索能力 + 频率分配"是否有交互效应

## 方案 B：纯 PPL 实验，更大 L_train

**想法**：不纠结 passkey，直接用 PPL 作为唯一指标，但换不同的 L_train 验证 τ* scaling law。

**实验设计**：
- [ ] 350M, base=500K, 50M tokens
- [ ] L_train=1024, τ*=64/√1024=2.0 → 跑 EVQ(τ=2.0) vs Geo
- [ ] L_train=4096, τ*=64/√4096=1.0 → 跑 EVQ(τ=1.0) vs Geo
- [ ] 只评 PPL@{L_train, 2×, 4×, 8×}

**优点**：直接验证 τ* scaling law 在不同 L_train 下的预测
**缺点**：跟 R6000 做的事有重叠（R6000 也有 L=4096 组）

## 方案 C：增加模型规模到 ~500M

**想法**：350M 太小，信噪比低。在 5090 32GB 上能跑 ~500M 模型。
