## 🎯 核心要求

### 1. 实验可信度分级（必须严格遵守）

| 等级 | 含义 | 引用状态 |
|------|------|----------|
| ✅ Tier 1 | 从零训练、多规模验证 | **论文主线** |
| ✅ Tier 2 | 机理实验、推理侧验证 | 论文支撑 |
| ⏳ Tier 3 | 进行中、待验证 | 暂不引用 |
| ⚠️ Tier 4 | 已知问题、需重跑 | **禁止引用** |
| ❌ 废弃 | 严重缺陷 | **禁止引用** |

### 2. 已验证的核心结论（可在论文中使用）

1. **多规模从零训练**：50M/100M/350M 跨规模一致改善 5%-14%
2. **Phase Collision 机理**：崩溃比从 22x 降至 1.08x
3. **注意力距离幂律**：D(Δ) ∝ Δ^(-γ)，γ≈0.72，R²=0.95
4. **理论相变点**：L/b>100 时 hybrid 优于 geo

### 3. 已知问题实验（禁止引用）

- **旧 8B LoRA 实验**：YaRN/PI 用 `rope_scaling`，Hybrid 用 monkey patch → 不公平
- **Qwen Hybrid-LoRA**：同样存在公平性问题
- **base=300k 消融**：Sigmoid 劣于 Standard（负结果）

### 4. 暂不宜宣传的结论

- "所有协议下都优于 YaRN/PI"
- "无需训练可直接替换频率"
- "Sigmoid 频率在所有 base 下都优于 Standard"

## 📝 优化任务

### 任务一：优化 `docs/`

1. **检查 EXPERIMENT_OVERVIEW.md**：
   - 确保所有实验都有可信度分级
   - 确保禁止引用的实验明确标注
   - 补充缺失的实验分析

2. **清理冗余**：
   - 合并重复内容
   - 删除过时信息
   - 确保单一信息源

3. **提高可读性**：
   - 添加导航索引
   - 使用表格总结关键数据
   - 添加推荐阅读顺序

### 任务二：优化 `knowledge_base/`

1. **检查一致性**：
   - 各文档之间的结论是否一致
   - 数字数据是否准确
   - 是否有遗漏的重要发现

2. **补充理论分析**：
   - 06_phase_collision_D_analysis.md 是否完整
   - 07_frequency_design_theory.md 是否需要更新
   - 08_8b_experiment_analysis.md 是否标记了公平性问题

3. **优化结构**：
   - 确保编号连贯
   - 添加交叉引用
   - 建立清晰的逻辑链

### 任务三：优化 `paper_draft/`

1. **确保论文主张与证据匹配**：
   - 只使用 Tier 1/Tier 2 证据
   - 明确标注哪些结论来自哪个实验
   - 不要引用 Tier 4 实验

2. **检查故事线**：
   - 问题 → 方法 → 证据 → 结论
   - 诚实披露边界条件和失败案例
   - 避免过度宣传

3. **补充缺失内容**：
   - 方法论部分是否完整
   - 实验设计是否清晰
   - 结果呈现是否规范

## ⚠️ 严格审核要求

对于每个实验，请检查：

1. **对照是否公平？**
   - 是否使用相同的训练超参？
   - 是否使用相同的 RoPE 实现方式？
   - 是否有 confounding factors？

2. **数据是否可靠？**
   - 是否有多次随机种子验证？
   - 是否报告了置信区间？
   - 是否有独立的复现验证？

3. **结论是否恰当？**
   - 是否有数据支持？
   - 是否过度外推？
   - 是否承认局限性？
